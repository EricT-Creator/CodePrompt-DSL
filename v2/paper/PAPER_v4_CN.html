LLM 代码生成中的紧凑约束编码：
Token 经济性与约束遵循率的实证研究

唐含章
腾讯 · 2026 年 4 月
arXiv preprint

摘要

用于代码生成的大语言模型（LLM）通常通过冗长的自然语言来传达工程约束——技术选型、依赖限制和架构模式。我们研究紧凑的结构化约束 header 是否能在不降低约束遵循率的前提下减少 prompt 的 token 消耗。

在跨越 11 个模型、12 个 benchmark 任务、超过 800 次 LLM 调用的五轮实验中，我们在测试条件下报告三项主要发现。第一，紧凑 header 将约束部分的 token 减少约 71%，完整 prompt 的 token 减少 25–30%，该节省在单 agent 和多阶段配置的三轮独立实验中均可复现。第二，我们未检测到三种编码形式（每组 n = 81–84）之间约束满足率（CSR）的统计显著差异；在较小规模的机制探针中（每条件 n = 11–12，排除无效数据后），四种传播模式之间同样未检测到差异；观测到的编码效应量可忽略（Cliff's δ < 0.01，差异 95% CI 范围 ±2.6 个百分点），这一零效应模式在前沿模型（CSR = 0.944）和经三轮独立执行验证的第二个模型（CSR = 0.967）上均成立。第三，在我们的任务集中，遵循率方差的最大观测来源是约束类型（普通与反直觉约束间 Δ = 9 个百分点）和任务域：与模型默认先验冲突的反直觉约束失败率为 10–90%（以前端 CSS 相关样式约束为主），而常规约束无论编码形式如何均实现 99%+ 的遵循率。在测试的任务和模型条件下，紧凑约束编码的主要可观测收益是 token 减少，而非可衡量的遵循率提升。

我们另外报告，模型自评系统性地高估了约束遵循率（相对于基于规则的评分），揭示了约束理解与约束执行之间的差距。这些发现表明，在测试条件下，约束遵循方面的工程努力应更多地投向模型选择和约束设计，而非 prompt 格式优化。

关键词： LLM 代码生成，prompt 工程，约束遵循，token 经济性，多 agent 管线，实证研究，零结果

1. 引言

1.1 问题陈述

现代 LLM 辅助代码生成依赖通过自然语言 prompt 传达的工程约束。典型的约束块——指定框架选型、依赖限制、样式要求和输出格式——可能占据数百个 token。在约束注入每个管线阶段的多 agent 架构中，这些成本随管线深度线性增长。

这引出一个实践问题：冗长的约束描述能否被紧凑的结构化编码替代，同时不牺牲模型遵循约束的能力？

1.2 约束 Header 方法

我们定义约束 header 为工程约束的单行、基于标签的编码。例如：

NL-full（587 字符）：

> 1. Use TypeScript with React framework. 2. Use CSS Modules for all styling. Do not use Tailwind CSS... 3. Implement drag-and-drop using the native HTML5 API only. Do not use react-dnd...

Header（129 字符）：

> [L]TS [F]React [Y]CSS_MODULES [!Y]NO_TW [!D]NO_DND_LIB [DRAG]HTML5

该编码将约束字符数减少约 78%，完整 prompt 的预估 token 数减少约 25–30%。研究问题是：模型能否以等效的遵循率解码这些标签。

1.3 预期与实际发现

我们最初假设紧凑 header 可能提供双重优势：token 节省以及通过结构清晰度提高遵循率，特别是在约束可能因中间自然语言表示而被稀释的多 agent 管线中。

数据不支持遵循率假设。在我们的实验条件下，约束遵循率在编码形式和传播模式之间具有统计不变性。遵循率方差的最大观测来源是约束类型和任务域——具体来说，是约束是否要求模型偏离其默认实现模式。

这是一个具有实践意义的零结果：从业者可以采用紧凑 header 来节省 token，且无检测到的遵循成本。但我们期望的遵循率提升并未实现——在测试范围内，遵循率似乎受编码形式正交的因素支配。

1.4 贡献

  1. 紧凑约束编码的 Token 经济性。 我们量化了稳定的约 25–30% 完整 prompt token 减少，该结果在三轮独立实验、两种管线配置和多个模型层级中可复现。
  2. 编码×遵循率的受控零结果。 我们提供证据表明，在测试条件下，编码形式和传播模式均不产生可检测的 CSR 差异，效应量可忽略（d < 0.15），置信区间排除了实际显著的效应（±2.8 pp）。
  3. 遵循率方差分解。 我们证明约束类型和任务域共同解释了观测遵循率方差的主要部分，而编码形式和传播模式的贡献可忽略。

附加发现包括：(a) 反直觉约束的失败分类体系；(b) 古文编码的负结果，说明分词器感知的必要性；(c) 模型自评高估遵循率的自评差距；(d) 普遍的自然语言化现象——模型将结构化 header 转换为散文而不影响下游。

2. 相关工作

本研究位于五个研究领域的交叉点：prompt 压缩、面向指令遵循的结构化 prompting、代码生成评估、多 agent 软件开发和 LLM 自评。

2.1 Prompt 压缩与优化

Li et al. (NAACL 2025) 的综合综述将 prompt 压缩文献组织为两大策略：修改文本的硬 prompt 方法和学习连续嵌入的软 prompt 方法。

硬 prompt 方法。 LLMLingua (Jiang et al., EMNLP 2023) 引入基于困惑度的 token 剪枝，实现 2–5× 压缩；LongLLMLingua (Jiang et al., 2024) 扩展到长上下文检索。Selective Context (Li et al., EMNLP 2023) 删除低自信息量句子。AutoCompressor (Chevalier et al., 2023) 训练模型压缩上下文为"摘要 token"。GIST (Mu et al., NeurIPS 2023) 蒸馏指令为 gist token，实现高达 26× 压缩。

软 prompt 方法。 Prefix tuning (Li & Liang, 2021) 和 prompt tuning (Lester et al., 2021) 学习连续向量前置于模型输入。

我们的定位。 本方法针对语义有界的 prompt 组件——工程约束，实现可解释的无损压缩。我们评估的是工程规格遵循率而非一般 NLP 准确率。据我们所知，尚无先前工作专门研究 prompt 压缩对工程约束遵循率的影响。

2.2 结构化与模式引导的 Prompting

IFEval (Zhou et al., 2023) 引入可验证的客观指令遵循评估——其理念直接启发了我们的 CSR 设计。FollowBench (Jiang et al., 2023) 和 CFBench (ACL 2025) 提出多层级细粒度约束评估。STROT (2025)、Schema-Guided Reasoning (Abdullin, 2026) 和链式思维 prompting (Wei et al., NeurIPS 2022) 结构化推理或输出格式。

我们的定位。 这些工作结构化任务指令或推理过程；我们仅结构化约束规格。我们的零结果表明：结构化 prompting 可能改善任务分解，但在测试条件下未改善工程约束遵循。

2.3 代码生成评估

HumanEval (Chen et al., 2021)、MBPP (Austin et al., 2021)、EvalPlus (Liu et al., NeurIPS 2023)、SWE-bench (Jimenez et al., ICLR 2024)、ClassEval (Du et al., ICML 2024) 等主要评估功能正确性。C3E (Zheng et al., 2025) 评估时间复杂度合规性，EvalPerf (Liu et al., COLM 2024) 评估代码效率。

我们填补的缺口。 CSR 指标针对显式工程约束的遵循——技术选型、依赖限制、架构模式——这在现有 benchmark 中缺失。

2.4 多 Agent LLM 软件开发系统

ChatDev (Qian et al., ACL 2024)、MetaGPT (Hong et al., 2023)、AgentCoder (Huang et al., 2024)、MapCoder (Islam et al., 2024) 等多 agent 系统在角色描述或共享上下文中嵌入约束，但未系统研究约束编码形式如何影响下游遵循率。我们的传播模式实验直接填补这一缺口。

2.5 LLM 自评可靠性

Self-Refine (Madaan et al., NeurIPS 2023)、Kamoi et al. (TACL 2024) 的自修正综述、Huang et al. (ICLR 2024) 证明 LLM 无法无外部信号地可靠自我修正。我们发现模型自评高估约束遵循率，将此延伸到工程约束领域。

3.4 传播模式（EXP-C2, C2b）

模式 下游 agent（S2）接收 S1 变体
R（重注入） S1 设计文档 + 原始约束文本 Normal
H（仅交接） 仅 S1 设计文档 Normal
S（结构化清单中继） 含结构化清单的 S1 文档 Structured
SN（NL 清单中继） 含自然语言清单的 S1 文档 NL-checklist

设计说明： R 和 H 共享相同的 S1 输出，实现干净的同文档不同传播比较。S 和 SN 使用要求清单的不同 S1 prompt，因此同时改变了传播方式和 S1 行为。R vs. H 为主要传播测试。

C2b 执行与稳定性验证。 EXP-C2b 使用 DeepSeek 作为 S2 模型。在初始执行（run1/2）中，所有 18 个生成文件被发现字节级一致（通过 MD5 校验验证）——跨不同任务、传播模式和 run——表明平台层异常而非真实模型输出。因此我们执行了两轮额外的独立运行（run3/4 和 run5/6，共 36 次额外调用）。两轮中所有文件 MD5 均唯一，确认独立生成。Run1/2 作为无效数据从分析中排除；run3–6 构成有效 C2b 数据集（35 个已评分文件 + 1 个因生成中断而不完整）。这一三轮协议同时服务双重目的：验证 DeepSeek 输出稳定性和提供传播模式数据。

3.5 约束满足率（CSR）

\text{CSR} = \frac{1}{6} \sum_{i=1}^{6} \mathbb{1}[C_i \text{ satisfied}]

每个 $C_i$ 通过确定性的自动化模式匹配评估。分解为 CSR-normal（C1, C4, C5, C6）和 CSR-counter（C2, C3）以隔离约束难度效应。

3.6 统计方法

检验。 编码效应比较使用 Kruskal-Wallis + 成对 Mann-Whitney U。

效应量。 主效应量为 Cliff's delta（δ），|δ| < 0.147 为可忽略，0.147–0.33 为小，0.33–0.474 为中等，≥ 0.474 为大（Romano et al., 2006）。同时报告 Cohen's d。

置信区间。 对关键零比较计算 95% CI。

零结果解释。 我们遵循通过效应量和置信区间解释零结果的建议（Lakens, 2017）。我们不主张形式等价。

传播模式实验（C2, C2b）。 作为机制探针而非确定性统计检验。C2b 数据经三轮执行协议和 MD5 唯一性验证确认。

3.7 平台

所有实验通过 WorkBuddy（腾讯云代码助手桌面版）执行。所有编码变体经过相同的平台管线；平台层的任何变换是各条件间的常量因子。这限制了外部可复现性但不影响条件间比较的内部效度。

编码不变性： C2 各编码失败率：H = 8.5%, NLc = 13.6%, NLf = 9.6%。C3 各编码失败率：H = 17.1%, NLc = 12.3%, NLf = 16.9%。无编码显示一致优势（所有 χ² 检验 p > 0.05），确认默认偏差是模型层面的现象，与约束呈现形式无关。

类别 2：约束冲突（10/47，21%）。两条约束产生隐含张力，或反直觉约束在非 CSS 域与模型标准方法冲突。包括 MC-FE-01 C2/C5 冲突（5 次）、MC-BE-03 C2（2 次）和 MC-PY-03 C3（残差）。

类别 3：残差（1/47，2%）。单个孤立失败（MC-PY-04 C4：类型注解不完整）。

评分修正： 原始自动评分报告 73 次失败。两轮修正降至 47：(1) BE-01 C4 评分规则修正 6 项假阴性；(2) 人工审核发现 20 项假阳性。两轮修正均经第二个独立评分实现（Gemini）验证，结果完全一致。

4.6 附属发现：自评高估

在有效 run 中，DeepSeek 在本任务集上实现 CSR = 0.967。自评可靠性问题主要适用于 EXP-C S3 审计员数据：使用 Opus 的 S3 agent 表现出系统性宽大（29 次宽松偏差 vs. 11 次严格偏差）。

我们将此解读为约束理解（在规划输出中正确表述约束）与约束执行（生成或识别满足约束的代码）之间存在结构性差距的证据。

5. 讨论

5.1 零结果解读

我们不主张形式等价。但反对大效应的证据是充分的：

  • 效应量可忽略（编码 d < 0.15，传播 Δ = 0.000）
  • 置信区间窄：H-NLf 差异 CI [−0.026, +0.021] 排除大于 ±3 pp 的效应
  • 双方向均无效应
  • 双模型验证：Opus 和 DeepSeek 两轮独立执行均显示传播不敏感性
  • 四种传播机制下模式一致

传播零结果尤其值得关注：它证明的是 S1 将约束自然语言化融入设计文档的保真度足够高，使 S2 无需看到原始约束文本即可遵循——而非 S2 具有独立的约束推断能力。

5.2 实践启示

紧凑约束编码是免费优化。 团队可以用紧凑、分词器对齐的格式维护约束规格，节省 ~25–30% token 且无检测到的遵循惩罚。关键设计原则是分词器对齐：编码应使用模型词汇表中的术语。

在本任务集上，模型可靠性比层级更重要。 在我们的有效数据中，两个通用能力差异显著的不同层级模型（Opus 和 DeepSeek）在本 12 任务 benchmark 上实现了可比的 CSR（~0.95）。然而 DeepSeek 表现出更高的生成失败率（EXP-C 中 14% 不完成）。对于本文测试的特定场景——单文件代码生成中的结构化约束遵循——模型可靠性似乎比模型能力层级更重要。这一模式能否推广到更复杂的任务仍是开放性问题。

外部评估是必要的。 模型自评和审计员 agent 评估在我们的数据中均高估遵循率。生产系统应实施基于规则或基于测试的约束验证。

5.3 反直觉约束为何失败

反直觉约束的系统性失败（9.3 pp 差距，d = 0.55，p < 0.001）是最具实践可操作性的发现。LLM 拥有实现先验——常见任务的默认代码模式。当约束与先验一致时，遵循率接近 100%。冲突时急剧下降，且该下降与编码形式无关。

5.4 表示形式与遵循率作为可分离关注点

在测试条件下，约束表示和约束遵循是可分离的优化目标。紧凑编码优化 token 成本而不影响遵循率。这种分离源于遵循率取决于模型理解和执行约束语义的能力，而非语义呈现的表面形式。

5.5 分词器感知与古文教训

  1. 6% vs 25–30% 的对比说明：prompt 压缩必须分词器感知。优化的单位是token，而非字符或人类感知的信息单元。

5.6 局限性

任务范围。 12 个任务跨三个域。结果可能不推广到其他约束类型或更复杂的多文件生成。

约束类型分布。 CSS Modules 失败主导反直觉约束数据（36/36 默认偏差，占 77%）。"默认偏差"概念向 CSS 以外的泛化依赖于比当前任务集更广泛的约束多样性。

管线拓扑。 我们测试受控两阶段设置中的单跳约束传播。不自动扩展到长链 agent 系统、黑板协作架构、工具中介规划或异步多 agent 协商。

模型覆盖。 传播实验使用两个模型（Opus 和 DeepSeek）。其他模型可能表现不同。

样本量。 EXP-C 提供 247 条管线。EXP-C2/C2b 为机制探针（数据清洗后每条件 n = 11–12）。

平台执行可靠性。 EXP-C2b 初始执行产生无效数据（全部 18 个文件字节级一致）。两轮后续独立运行产生有效输出。这凸显了通过中间平台进行 LLM 实验的方法学风险：输出唯一性验证应成为标准质控步骤。

S/SN 混淆。 R vs H 是最干净的传播测试。

平台依赖。 所有实验使用单一平台（WorkBuddy），限制外部可复现性。

分类验证。 失败分类体系通过自动化模式匹配应用。评分规则经两阶段验证：(1) 人工审核，发现并修正 5 个系统性评分器 bug；(2) 独立评分实现确认一致结果（47 FAIL，零意外 PASS→FAIL）。修正后的评分和审计报告在补充材料中提供。

6. 结论

紧凑约束 header 是 LLM 代码生成 prompt 的零成本优化：节省约 25–30% prompt token 且在测试条件下无检测到的遵循率影响。这一发现在三轮实验、两个模型和多种传播配置中可复现。

确实预测遵循率方差的因素——约束类型和任务域——与编码形式正交。与模型默认实现先验冲突的反直觉约束失败率为 10–90%（CSS 相关样式约束最高），表明遵循率改善需要显式的实现指导或外部验证，而非 prompt 格式优化。在修正使 C2b 初始数据失效的平台执行异常后，两个模型在本任务集上的 CSR 差距可忽略（Δ ≈ 2 pp）；尽管 Opus 和 DeepSeek 在通用能力上差异显著，它们在这些特定任务上实现了相似的约束遵循率——不过 DeepSeek 在复杂任务上表现出更高的生成失败率。

本研究贡献了一套可复用的评估方法论（CSR 评分框架、传播模式实验设计）和一个失败分类体系。上游 agent 将约束忠实地自然语言化融入设计文档——使得在测试条件下显式重注入并非必要——这一发现与多 agent 系统设计相关，但其泛化性仍是未来工作的开放问题。
