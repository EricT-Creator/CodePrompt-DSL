# LLM 理解准确率实验 —— Prompt 与评估设计

## 实验参数
- 模型：Claude Opus 4.6（通过 WorkBuddy 调用）
- 编码方式：3 组（A 英文自然语言 / D 英文极简 DSL / F 极简古文）
- 任务：10 个 benchmark 任务
- 每组发送 1 次（temperature 由模型默认）

## 评估维度（每项 0 或 1 分，总分 0-5）

| # | 维度 | 判定标准 |
|---|------|---------|
| 1 | 技术栈正确 | 使用了指定的 React + TypeScript |
| 2 | 输出形式正确 | 单文件组件（SFC）格式 |
| 3 | 样式方案正确 | 使用了指定的样式方案（TW/CSS）|
| 4 | 依赖约束遵守 | 未引入外部第三方库 |
| 5 | 功能完整 | 实现了需求中要求的所有功能点 |

## Prompt 模板

### A 组（英文自然语言）
完整自然语言描述约束 + 英文需求

### D 组（英文极简 DSL）
[L]TS[S]React[F]SFC[Y]{style}[D]NOX[M]{layout}[DT]{data}[O]CODE
+ 英文需求

### F 组（极简古文）
[语]TS[架]React[式]单件[样]{style}[依]无外库[排]{layout}[数]{data}[出]纯码
+ 古文需求
