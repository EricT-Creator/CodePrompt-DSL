## 三、即便没有源码，我仍然能验证的部分

## 3.1 评估器确实存在“字面关键词偏置”

从 `evaluate_claude_haiku.py` 可以确认，当前评估逻辑本质上是**字面字符串匹配**：

### 技术栈检测
- 直接检查：`import React` 是否出现
- 直接检查：`: React.FC` 是否出现

这意味着：
- 如果代码用了新的 JSX runtime，不写 `import React`，可能被误判
- 如果代码用了 TypeScript，但没有写成 `: React.FC` 风格，也可能被误判成 `TS:false`

### 功能检测
- `T07` 只看：`grid`, `lightbox`, `lazy`
- `T09` 只看：`sort`, `paginat`, `search`
- 达到 75% 才给 1 分

这意味着：
- 如果图库实现用的是 `modal` / `overlay` / `preview`，但没写 `lightbox`，就会丢分
- 如果分页实现用的是 `page` / `pageSize` / `currentPage`，但没写 `paginat` 前缀，也会丢分

**这点是可以正式确认的，不是猜测。**

所以：

> 当前评估体系对“功能是否真的实现”并不直接判断，而是高度依赖“是否用了预设词”。

这给 H1 提供了很强的机制支持。

## 四、三条猜想的重新判定

## H1「词汇选择偏好假说」

### 原假说
> 模型之间的得分差异，主要不是能力差距，而是词汇选择偏好与评估关键词之间的匹配程度。

### 现在能确认到哪一步？

**结论：强烈可疑，但尚未被人工复核证实。**

### 支持证据
1. `T07` / `T09` 在多个强模型上重复出现相同模式：
   - GLM：T07 = `grid,lazy`，T09 = `sort,search`
   - Kimi：T07 = `grid,lazy`，T09 = `sort,search`
   - MiniMax：T09 = `sort,search`
   - GPT / Gemini-Pro / Gemini-Flash：则能稳定命中 `lightbox` / `paginat`

2. 评估器本身确实只认字面词，不认等价表达。

### 但为什么还不能直接宣布成立？
因为**可信模型的源码不在仓库里**，我看不到它们到底有没有实现：
- T07 的点击放大预览
- T09 的实际分页逻辑

所以我现在不能严谨地说：
- “它其实做了，只是没写到关键词”

我只能说：
- “当前评分机制确实会把这类情况判成低分，而且多个模型的失分模式高度像这种情况”

### 当前判定
- **机制层：已证实存在偏差**
- **案例层：尚未完成人工代码复核**
- **结论层：暂不能写成‘已验证成立’，只能写成‘高概率成立，待源码复核确认’**

---

## H2「能力阈值假说」

### 原假说
> 模型要么跨过 React+TS 范式阈值拿到 4.3+，要么只能拿 1.x，中间不存在渐变。

### 这条需要修正

**结论：模型均分确实呈现双峰，但“能力全有或全无”这个说法过头了。**

### 支持“双峰”的证据
从模型均分看，的确分成两堆：
- 强模型：约 4.3 ~ 5.0
- 弱模型：约 1.1 ~ 1.3

### 但反证也很明显
#### DeepSeek-V3.2
- `T01` 能拿到 `4/5`
- 说明它并不是所有任务都“完全不会”
- 它至少能在某个任务上满足 React、TS、export、功能词，只是没过样式

#### Gemini-3.1-Flash-Lite
- 所有任务几乎都是：`React:true TS:false`、`export:false`
- `T06` 还能拿到 `2/5`
- 说明它不是完全空白输出，而是输出了某种“半成品 React 风格代码”

### 因此更准确的表述应该是
> **当前实验的“模型平均分”呈现双峰分布，但任务级行为并不是纯粹的两态开关。弱模型不是完全不会写，而是常常卡在 TypeScript 显式写法、`export default` 形式约束、Tailwind 命中等结构适配层。**

### 当前判定
- **“均分双峰”成立**
- **“能力全有或全无”不成立，或至少证据不足**
- **应改写为“结构化工程约束适配能力存在明显分层”**

## H3「古文免疫假说」

### 原假说
> 主流模型已经对 Prompt 表面编码形式高度免疫，古文衰减不是普遍现象，而是个别模型现象。

### 现在能确认到哪一步？

**结论：结果层面基本成立，但原因层面还没被严格验证。**

### 可以确认的结果事实
从 JSON 可以直接确认：
- GPT-5.4：A = D = F
- Gemini-3.0-Pro：A = D = F
- Gemini-3.0-Flash：A = D = F
- DeepSeek-V3.2：A = D = F
- Gemini-3.1-Flash-Lite：A = D = F
- GLM / Kimi：仅 F 组轻微下降
- Claude Opus：F 组下降最明显

所以“**古文衰减不是所有模型的共同现象**”这点，本身是成立的。

### 但不能确认的地方
Claude Opus 的 F 组下滑，到底是：
1. 真的没理解古文需求；还是
2. 做出来了，但没命中关键词；还是
3. 两者都有；

当前没有源码，不能逐文件确认。

### 当前判定
- **结果统计层：成立**
- **机制解释层：未完全验证**
- **对外表述宜改成：‘古文衰减并非普遍现象，但其成因仍需源码复核确认。’**
